# 電卓アプリケーション 要求仕様書

## 1. プロジェクト概要

### 1.1 プロジェクト名
SWA Calculator（Azure Static Web Apps 電卓アプリケーション）

### 1.2 目的
React を使用したモダンな Web 電卓アプリケーションを構築し、Azure Static Web Apps にデプロイする。
基本的な四則演算に加え、計算履歴とメモリ機能を備えた実用的な電卓を提供する。

### 1.3 対象ユーザー
一般ユーザー（PC、タブレット、スマートフォンから利用）

---

## 2. 機能要件

### 2.1 基本演算機能
- **四則演算**：加算（+）、減算（-）、乗算（×）、除法（÷）
- **数値入力**：0-9 の数字入力、小数点入力
- **クリア機能**：
  - C（クリア）：現在の入力をクリア
  - AC（オールクリア）：すべての計算状態をリセット
- **等号**：計算結果の表示（=）

### 2.2 メモリ機能
- **複数メモリスロット**：3-5 個のメモリスロットを提供
- **メモリ操作**：
  - M+（メモリ加算）：現在の値を選択中のメモリスロットに加算
  - M-（メモリ減算）：現在の値を選択中のメモリスロットから減算
  - MR（メモリ呼出）：選択中のメモリスロットの値を呼び出し
  - MC（メモリクリア）：選択中のメモリスロットをクリア
  - メモリスロット選択：複数のスロット間を切り替え可能
  - 各スロットの値を一覧表示（UI上で確認可能）

### 2.3 計算履歴機能
- **履歴の記録**：各計算式と結果を記録
  - 形式例：`123 + 456 = 579`
- **履歴の表示**：計算履歴を時系列で表示（新しいものが上）
- **履歴の上限**：1000 件まで保存
  - 上限超過時：最も古い履歴から自動削除（FIFO方式）
- **履歴の操作**：
  - 履歴項目をクリックして値を再利用
  - 履歴全体のクリア機能
  - 個別履歴の削除機能

### 2.4 データ保存・エクスポート機能
- **セッション中の保存**：
  - すべてのデータ（計算履歴、メモリスロット）はブラウザのメモリ上に保持
  - ページリロード時にデータは消失
- **エクスポート機能**：
  - **JSONファイルダウンロード**：計算履歴とメモリデータを JSON 形式でダウンロード
  - **クリップボードコピー**：計算履歴をテキスト形式でコピー
- **インポート機能**（オプション）：以前ダウンロードした JSON ファイルを読み込んで履歴を復元

### 2.5 キーボード操作機能
- **数字キー**：0-9
- **演算子キー**：+、-、*、/
- **制御キー**：
  - Enter：計算実行（= と同じ）
  - Backspace/Delete：最後の1文字を削除
  - Escape：クリア（C と同じ）
  - Ctrl+Shift+Delete：オールクリア（AC と同じ）
- **その他**：
  - .（ピリオド）：小数点入力
  - キーボードショートカット一覧を表示する機能（?キーまたはヘルプボタン）

## 3. 非機能要件

### 3.1 技術スタック
- **フロントエンド**：
  - React 18+
  - TypeScript（型安全性のため推奨）
  - Vite（ビルドツール）
  - CSS Modules または Styled Components（スタイリング）
- **バックエンド**：なし（フロントエンドのみの静的アプリケーション）
- **ホスティング**：Azure Static Web Apps
- **CI/CD**：GitHub Actions

### 3.2 UI/UX要件
- **レスポンシブデザイン**：
  - デスクトップ（1024px 以上）
  - タブレット（768px - 1023px）
  - スマートフォン（320px - 767px）
- **デザインスタイル**：
  - シンプルで直感的なデザイン
  - 一般的な電卓アプリケーションの UI パターンを踏襲
  - ボタンの視覚的フィードバック（ホバー、アクティブ状態）
- **アクセシビリティ**：
  - キーボードナビゲーション対応
  - ARIA ラベルの適切な使用
  - 適切なカラーコントラスト

### 3.3 ブラウザ対応
- **対象ブラウザ**：
  - Chrome（最新版）
  - Firefox（最新版）
  - Safari（最新版）
  - Edge（最新版）
- **非対応**：IE11

### 3.4 パフォーマンス
- **初回ロード時間**：3秒以内
- **操作レスポンス**：100ms 以内
- **バンドルサイズ**：500KB 以下（gzip圧縮後）

### 3.5 セキュリティ
- **データ保存**：クライアントサイドのみ（サーバーへの送信なし）
- **XSS対策**：React の標準的なエスケープ処理を使用

## 4. デプロイ要件

### 4.1 Azure Static Web Apps
- **環境**：
  - 本番環境（main ブランチ）
  - プレビュー環境（プルリクエスト毎に自動生成）

### 4.2 GitHub Actions
- **自動デプロイフロー**：
  1. main ブランチへの push をトリガー
  2. ビルド実行（`npm run build`）
  3. Azure Static Web Apps へデプロイ
  4. デプロイ完了通知
- **プルリクエスト時**：
  - プレビュー環境を自動作成
  - プレビュー URL をコメントに追加

### 4.3 環境変数
- 特になし（静的アプリケーションのため）

---

## 5. 制約事項・前提条件

### 5.1 制約事項
- データの永続化は行わない（セッションベース）
- 複雑な科学計算機能は含まない（四則演算のみ）
- 複数ユーザー間でのデータ共有機能はなし

### 5.2 前提条件
- GitHub アカウントを保有していること
- Azure アカウントを保有していること
- Node.js 18+ がインストールされていること

## 6. 開発スコープ

### 6.1 Phase 1（MVP）
- 基本的な四則演算機能
- 1つのメモリスロット（M+, M-, MR, MC）
- 基本的な計算履歴（100件程度）
- キーボード操作（基本的な数字と演算子）
- シンプルな UI デザイン
- Azure Static Web Apps へのデプロイ

### 6.2 Phase 2（機能拡張）
- 複数メモリスロット（3-5個）への拡張
- 計算履歴の上限を1000件に拡張
- 履歴のダウンロード/コピー機能
- 拡張キーボードショートカット
- レスポンシブデザインの最適化
- UI/UX の改善

### 6.3 Phase 3（オプション機能）
- 履歴のインポート機能
- テーマ切替機能（ライト/ダーク）
- 計算履歴の検索/フィルタ機能
- PWA 対応（オフライン利用）
